# Legal & Shariah AI Agent

AI-помощник для юридического анализа документов по законодательству Республики
Узбекистан и анализа документов по исламскому банкингу.

> ⚠️ **Дисклеймер.** Ответы AI-агента являются аналитической и справочной
> информацией. Они не являются юридическим заключением, адвокатской
> консультацией, официальной позицией регулятора или шариатского совета. Перед
> принятием решений требуется проверка квалифицированным юристом и, при
> необходимости, заключение шариатского совета.

---

## Что умеет

- Загрузка документа (**TXT, DOCX, PDF**) или вставка текста вручную.
- 6 режимов анализа:
  1. Юридический анализ (Республика Узбекистан)
  2. Банковский юридический анализ
  3. Шариатский анализ
  4. Анализ исламского банковского продукта
  5. Краткий комментарий к пункту
  6. Новая редакция пункта
- Структурированный результат с таблицами рисков и кнопкой «Скопировать».
- Все вызовы OpenAI выполняются на сервере — **API-ключ не попадает в браузер**.

---

## Технологии

Next.js (App Router) · TypeScript · Tailwind CSS · OpenAI Node SDK (Responses
API) · mammoth (DOCX) · pdf-parse (PDF).

База данных не используется. Документы не сохраняются на сервере, полный текст не
логируется.

## Переменные окружения

Скопируйте `.env.example` в `.env.local` и впишите свой ключ:

```
OPENAI_API_KEY=ваш_ключ_openai
OPENAI_MODEL=gpt-5.5
```

- `OPENAI_API_KEY` — **обязателен**. Без него анализ работать не будет (сайт
  покажет понятное сообщение).
- `OPENAI_MODEL` — необязателен. Если не задан, используется `gpt-5.5`.

`.env.local` уже добавлен в `.gitignore` и **никогда не попадёт в GitHub**.

## Запуск на компьютере (локально)

> Нужен установленный **Node.js 18.18+** (рекомендуется LTS 20).
> Скачать: <https://nodejs.org> → кнопка «LTS». Установите и перезапустите
> терминал.

1. Установите зависимости:
   ```bash
   npm install
   ```
2. Создайте файл `.env.local` (скопируйте из `.env.example`) и впишите ключ.
3. Запустите сайт:
   ```bash
   npm run dev
   ```
4. Откройте в браузере: <http://localhost:3000>
   Страница анализа: <http://localhost:3000/analyze>

Чтобы остановить — нажмите `Ctrl + C` в терминале.

## Деплой на Vercel

1. Зайдите на <https://vercel.com> и войдите через GitHub.
2. Нажмите **Add New… → Project** и выберите ваш репозиторий.
3. Vercel сам определит Next.js — настройки сборки менять не нужно.
4. Откройте раздел **Environment Variables** и добавьте:

   | Name             | Value                     |
   | ---------------- | ------------------------- |
   | `OPENAI_API_KEY` | ваш реальный ключ OpenAI  |
   | `OPENAI_MODEL`   | `gpt-5.5` (необязательно)  |

5. Нажмите **Deploy** и дождитесь окончания сборки.
6. Получите ссылку вида `https://ваш-проект.vercel.app`.

После каждого `git push` в ветку `main` Vercel автоматически пересоберёт сайт.

## Безопасность и конфиденциальность

- API-ключ хранится только в переменных окружения и используется **только на
  сервере** (`src/app/api/...`). В браузер он не передаётся.
- Полный текст загруженных документов **не логируется** (в логи попадают только
  длина текста и тип ошибки).
- Документы **не сохраняются** после анализа (в MVP нет базы данных).
- **Для банковской версии** требуется отдельный приватный деплой (private
  deployment), договор об обработке данных с OpenAI / собственная модель и
  отдельная политика обработки персональных и банковских данных. Это отмечено
  комментариями в коде (`src/app/api/analyze/route.ts`).
