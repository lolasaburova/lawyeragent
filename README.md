# Legal & Shariah AI Agent

AI-помощник для юридического анализа документов по законодательству Республики
Узбекистан и анализа документов по исламскому банкингу.

> ⚠️ **Дисклеймер.** Ответы AI-агента являются аналитической и справочной
> информацией. Они не являются юридическим заключением, адвокатской
> консультацией, официальной позицией регулятора или шариатского совета. Перед
> принятием решений требуется проверка квалифицированным юристом и, при
> необходимости, заключение шариатского совета.

---

## Что умеет

- Загрузка документа (**TXT, DOCX, PDF**) или вставка текста вручную.
- 6 режимов анализа:
  1. Юридический анализ (Республика Узбекистан)
  2. Банковский юридический анализ
  3. Шариатский анализ
  4. Анализ исламского банковского продукта
  5. Краткий комментарий к пункту
  6. Новая редакция пункта
- Структурированный результат с таблицами рисков и кнопкой «Скопировать».
- Все вызовы OpenAI выполняются на сервере — **API-ключ не попадает в браузер**.

---

## Технологии

Next.js (App Router) · TypeScript · Tailwind CSS · OpenAI Node SDK (Chat
Completions API) · PostgreSQL + Prisma · mammoth (DOCX) · pdf-parse (PDF).

Хранение в PostgreSQL: история анализов и чатов (как в ChatGPT) и база
юридических источников для цитирования. Полный текст документов и история чата
не логируются.

## Переменные окружения

Скопируйте `.env.example` в `.env.local` и впишите свой ключ:

```
OPENAI_API_KEY=ваш_ключ_openai
OPENAI_MODEL=gpt-5.5
```

- `OPENAI_API_KEY` — **обязателен**. Без него анализ работать не будет (сайт
  покажет понятное сообщение).
- `OPENAI_MODEL` — необязателен. Если не задан, используется `gpt-5.5`.

`.env.local` уже добавлен в `.gitignore` и **никогда не попадёт в GitHub**.

## Запуск на компьютере (локально)

> Нужен установленный **Node.js 18.18+** (рекомендуется LTS 20).
> Скачать: <https://nodejs.org> → кнопка «LTS». Установите и перезапустите
> терминал.

1. Установите зависимости:
   ```bash
   npm install
   ```
2. Создайте файл `.env.local` (скопируйте из `.env.example`) и впишите:
   - `OPENAI_API_KEY` — ваш ключ OpenAI;
   - `OPENAI_MODEL` — `gpt-4o-mini` (по умолчанию);
   - `lawyer_DATABASE_URL` (или `DATABASE_URL`) — строка подключения к
     PostgreSQL (см. раздел «База данных»).
3. Примените схему БД (создаст таблицы):
   ```bash
   npm run prisma:migrate
   ```
4. Запустите сайт:
   ```bash
   npm run dev
   ```
5. Откройте в браузере: <http://localhost:3000>
   Страница анализа: <http://localhost:3000/analyze> · База источников: <http://localhost:3000/sources>

Чтобы остановить — нажмите `Ctrl + C` в терминале.

> Без `DATABASE_URL` анализ и чат всё равно работают (без сохранения истории):
> боковая панель просто покажет, что история недоступна.

## База данных (PostgreSQL + Prisma)

История анализов, чаты и база юридических источников хранятся в PostgreSQL через
Prisma. Подойдёт любой PostgreSQL: локальный, **Vercel Prisma Postgres**,
**Neon**, **Supabase** и т. п.

### Переменные окружения для БД

**На Vercel** интеграция Prisma Postgres автоматически создаёт переменные с
префиксом проекта, например:

- `lawyer_DATABASE_URL`
- `lawyer_POSTGRES_URL`
- `lawyer_PRISMA_DATABASE_URL`

Этот проект использует на Vercel именно **`lawyer_DATABASE_URL`** — отдельно
создавать `DATABASE_URL` не нужно. Схема Prisma читает `lawyer_DATABASE_URL`
(см. `prisma/schema.prisma`), а во время выполнения клиент дополнительно делает
fallback на `DATABASE_URL` (см. `src/lib/prisma.ts`:
`process.env.lawyer_DATABASE_URL || process.env.DATABASE_URL`).

**Локально** можно задать любую из двух переменных в `.env.local`. Поскольку
Prisma CLI (`migrate`, `studio`) читает имя из схемы, для локальных миграций
проще задать `lawyer_DATABASE_URL`:

```
lawyer_DATABASE_URL=postgresql://user:password@localhost:5432/dbname?schema=public
# (опционально, fallback для рантайма)
DATABASE_URL=postgresql://user:password@localhost:5432/dbname?schema=public
```

### Команды

```bash
npm run prisma:generate   # сгенерировать клиент (выполняется и при npm install)
npm run prisma:migrate    # создать/применить миграцию (локально)
npm run prisma:studio     # визуальный редактор БД
npm run prisma:deploy     # применить миграции на проде (= prisma migrate deploy)
```

На Vercel схема применяется один раз (или при изменении схемы):

```bash
npx prisma migrate deploy
```

Таблицы: `LegalSource`, `LegalSourceChunk`, `ChatSession`, `ChatMessage`
(см. `prisma/schema.prisma`).

## Как добавить первый юридический источник

1. Откройте страницу **/sources**.
2. В блоке «Новый источник» укажите название, категорию (например `banking`),
   тип (`law`), при необходимости — URL, юрисдикцию, язык. Нажмите «Добавить
   источник».
3. У созданного источника заполните форму «Добавить фрагмент»: статья/пункт,
   заголовок, текст нормы. Нажмите «+ Добавить фрагмент».
4. Теперь при анализе и в чате AI сможет находить и цитировать этот фрагмент в
   разделе **«Источники и проверка»**. Если подходящего источника нет, AI явно
   пишет «Источник не подключен: требуется дополнительная проверка».

> Страница `/sources` — это MVP без аутентификации. Для продакшена защитите её
> авторизацией и ролевым доступом (только администратор).

## Деплой на Vercel

1. Зайдите на <https://vercel.com> и войдите через GitHub.
2. Нажмите **Add New… → Project** и выберите ваш репозиторий.
3. Vercel сам определит Next.js — настройки сборки менять не нужно.
4. Откройте раздел **Environment Variables** и добавьте:

   | Name                  | Value                                       |
   | --------------------- | ------------------------------------------- |
   | `OPENAI_API_KEY`      | ваш реальный ключ OpenAI                    |
   | `OPENAI_MODEL`        | `gpt-4o-mini` (необязательно)               |
   | `lawyer_DATABASE_URL` | строка подключения к PostgreSQL             |

   Проще всего подключить БД через **Storage → Prisma Postgres** в самом
   Vercel — он автоматически создаёт переменные с префиксом проекта, включая
   **`lawyer_DATABASE_URL`**. Отдельно создавать `DATABASE_URL` не нужно — проект
   использует `lawyer_DATABASE_URL`.

5. Нажмите **Deploy** и дождитесь окончания сборки (`prisma generate`
   выполняется автоматически на этапе `postinstall`).
6. Примените схему к боевой БД один раз (задав `lawyer_DATABASE_URL` на боевую
   базу — локально или через Vercel CLI):
   ```bash
   npx prisma migrate deploy
   ```
7. Получите ссылку вида `https://ваш-проект.vercel.app`.

После каждого `git push` в ветку `main` Vercel автоматически пересоберёт сайт.

## Безопасность и конфиденциальность

- API-ключ хранится только в переменных окружения и используется **только на
  сервере** (`src/app/api/...`). В браузер он не передаётся.
- Полный текст загруженных документов и история чата **не логируются** (в логи
  попадают только длина текста, число сообщений, модель и тип ошибки).
- **Документы и чаты теперь сохраняются в базе данных** (для истории как в
  ChatGPT). В интерфейсе показано предупреждение: не загружайте банковскую
  тайну, персональные данные или коммерческую тайну без разрешения.
- **Для банковского продакшена** добавьте аутентификацию, шифрование, контроль
  доступа, аудит-логи и политику хранения данных (data retention). Также нужен
  приватный деплой (private deployment), договор об обработке данных с OpenAI или
  собственная модель и отдельная политика обработки данных. Это отмечено
  комментариями в коде (`src/app/api/analyze/route.ts`, `src/app/sources/page.tsx`).
